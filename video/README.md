# Video Analysis Module

This module analyzes surveillance-style video clips and generates a structured event log CSV.

## Folder Layout

- `src/` contains the motion and detection script
- `data/` contains input video clips
- `frames/` stores sampled frames generated during processing
- `output/` contains the final event log CSV

## Supported Input

The current script reads `.mpg` files from `data/`.

Sample clips already included in this folder:

- `data/Browse1.mpg`
- `data/Fight_Chase.mpg`
- `data/LeftBag.mpg`
- `data/Rest_FallOnFloor.mpg`
- `data/Walk1.mpg`

## Output

The pipeline writes:

- `output/video_event_log.csv`

Current columns:

- `Clip_ID`
- `Timestamp`
- `Frame_ID`
- `Event_Detected`
- `Persons_Count`
- `Confidence`

The script also saves sampled frames under `frames/` for each processed clip.

## Setup

From the `video/` folder:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Model Weights

The script uses YOLOv8 person detection through Ultralytics.

Preferred setup:

- keep `yolov8n.pt` in the project root:
  `../yolov8n.pt`

The script first looks for a local `yolov8n.pt` file in the repository root. If it is missing, Ultralytics may try to download it automatically.

## Run

From the `video/` folder:

```bash
python3 src/motion_detection.py
```

The script will:

- read `.mpg` clips from `data/`
- sample frames at roughly 1-second intervals
- detect motion and persons
- infer conservative event labels
- save the event log to `output/video_event_log.csv`

## Event Labels

Current event labels include:

- `pedestrian appearance`
- `pedestrian presence`
- `stationary pedestrian`
- `pedestrian movement`
- `brisk pedestrian movement`
- `group presence`
- `active group movement`
- `high-intensity group activity`

Frames with only scene/background motion may be processed internally but are filtered out of the final event log unless they become meaningful tracked events.

## Notes

- This pipeline is rule-based and intentionally conservative.
- `frames/` contains intermediate artifacts and can be regenerated by rerunning the script.
- If no `.mpg` files are found in `data/`, the script exits without creating a new output file.
- The final integration pipeline links this output by `Clip_ID` through
  `integration/data/incident_map.csv`.
